My GitHub Terminal Guide

Quick notes on how to push your code to GitHub

Step 1: Initialize Git in your project

First, navigate to your project folder in the terminal. Then, tell Git to start tracking your project:

cd /path/to/your/project
git init

This creates a hidden `.git` folder in your directory.

Step 2: Add your files

Tell Git which files you want to include in your next commit. You can add all files, or specific ones:

git add .
// Adds all new and modified files to the staging area 
git add index.html style.css
// Adds specific files

The git add command moves files from your working directory to the staging area.

Step 3: Commit your changes

Now, save a snapshot of your staged files with a descriptive message:

git commit -m "Initial commit for my project"

The -m flag is for your commit message. Make it meaningful!

Step 4: Connect to GitHub (Remote Repository)

On GitHub, create a **new empty repository** (don't initialize with README). Copy the `HTTPS` URL. Then, connect your local project to it:

git remote add origin https://github.com/your-username/your-repo-name.git

origin is just a nickname for your remote repository. Replace the URL with yours!

Step 5: Push your local changes to GitHub

Finally, send your committed changes from your local machine to GitHub:

git push -u origin main

-u origin main sets the upstream branch, so next time you can just type git push. If your default branch is named master instead of main, use git push -u origin master.

That's it!

Your project should now be live on GitHub. Keep pushing updates regularly!

Good luck, programmer!
